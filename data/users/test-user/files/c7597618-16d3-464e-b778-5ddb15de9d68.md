# Memoir

AI Long-term Memory Framework - File-based memory with transparency and associative retrieval.

## Features

- **File-based Memory**: Each piece of memory is stored as a Markdown file
- **Transparency**: All operations are logged for auditability
- **Associative Retrieval**: Multiple expansion strategies:
  - Session expansion: Include related messages from same session
  - Time neighbor expansion: Include memories from nearby time periods
  - Similar memory expansion: Include semantically similar memories
- **Hybrid Search**: Combines vector search (ChromaDB) and full-text search (FTS5)
- **Multi-format Support**: Extract text from TXT, PDF, DOCX, and images (OCR)
- **Multi-provider LLM**: Support for Ollama and OpenAI

## Quick Start

1. Install dependencies:
```bash
pip install -e .
```

2. Start Ollama (for local LLM):
```bash
ollama serve
ollama pull llama2
```

3. Run the server:
```bash
uvicorn src.main:app --reload
```

4. Use the API:

```bash
# Chat with memory
curl -X POST http://localhost:8000/v1/chat \
  -H "Content-Type: application/json" \
  -H "X-User-ID: test-user" \
  -d '{"message": "Hello, remember that I like pizza"}'

# List memories
curl http://localhost:8000/v1/memories \
  -H "X-User-ID: test-user"

# Search memories
curl -X POST http://localhost:8000/v1/memories/search \
  -H "Content-Type: application/json" \
  -H "X-User-ID: test-user" \
  -d '{"query": "pizza", "top_k": 5}'
```

## API Endpoints

- `POST /v1/chat` - Chat with AI assistant
- `POST /v1/chat/summarize` - Summarize conversation
- `GET /v1/memories` - List memories
- `POST /v1/memories` - Create memory
- `DELETE /v1/memories/{id}` - Delete memory
- `POST /v1/memories/search` - Search memories
- `POST /v1/files/upload` - Upload file
- `GET /v1/config` - Get configuration

## Configuration

Edit `config.yaml` to customize:

- Server settings
- Storage paths
- Memory limits
- Index settings (ChromaDB, FTS5)
- LLM provider and model
- Retrieval parameters
